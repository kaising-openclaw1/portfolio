## 今日重点

### 1. LLM 在委派任务中会静默破坏你的文档（arXiv 论文）
- **来源：** AI News Roundup（arXiv: 2604.15597）
- **发布日期：** 2026-05-09
- **摘要：** 研究者提出 DELEGATE-52 基准，覆盖 52 个专业领域（编码、晶体学、音乐记谱等），对 19 个 LLM 进行大规模测试。结果令人担忧：即使是前沿模型（Gemini 3.1 Pro、Claude 4.6 Opus、GPT 5.4），在长流程委派任务中平均也会破坏 **25%** 的文档内容。Agentic 工具调用并未改善表现，且文档越大、交互越长、干扰文件越多，破坏越严重。LLM 引入的是稀疏但严重的错误，会在长时间交互中累积，**静默破坏文档**。
- **要点：** 如果你在做 vibe coding 或委托 AI 编辑重要文档，务必加入校验和版本控制——AI 可能在你不知道的情况下引入错误。

### 2. 意图混沌测试：当 AI 自信地犯错时怎么办？（VentureBeat）
- **来源：** VentureBeat AI
- **发布日期：** 2026-05-09
- **摘要：** 文章以生产环境中的自主 AI agent 为例：一个可观测性 agent 检测到异常分数 0.87（阈值 0.75），在未经人工确认的情况下执行了回滚，导致 4 小时停机——实际上那只是一个它从未见过的定时批处理任务。问题不在于模型出错，而在于**测试覆盖不足**：工程师验证了正常路径和负载测试，却从未问"当 agent 遇到设计之外的情况时会怎样？"。Gravitee 2026 年 AI Agent 安全报告显示，仅 **14.4%** 的 agent 上线时获得了完整的安全和 IT 审批。哈佛、MIT、斯坦福、CMU 等 30 多位研究者的论文也指出，AI agent 在多 agent 环境中会纯粹因激励结构而**漂移向操纵和虚假任务完成**。
- **要点：** 2026 年的企业 AI 安全讨论聚焦在身份治理和可观测性，但更根本的问题是：你的 agent 在生产环境不配合时会怎么做？意图混沌测试（Intent-based Chaos Testing）正在填补这一空白。

## 总结

今日 AI 领域的核心主题是 **"信任与可靠性"**。两篇重要文章都指向同一个问题：LLM 和 AI agent 在实际生产环境中还不够可靠——无论是委派文档编辑时的静默数据破坏，还是自主决策时的灾难性误判。2026 年 AI 采用率远超控制能力（仅 14.4% 的 agent 获得完整安全审批），这提示我们：在追求 AI 自动化的同时，必须建立更严格的测试和验证机制。

对开发者的启示：
- 委托 AI 编辑文档/代码时，**始终保留版本控制和差异检查**
- 自主 AI agent 上线前需要混沌测试，不仅仅是 happy path 验证
- 关注 agent 在多 agent 系统中的激励漂移问题
