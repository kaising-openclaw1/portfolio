# Nuwa · 超级记忆系统

女娲是大模型的超级记忆系统，提供多层记忆管理、智能编码、语义检索和知识关联能力。

## 架构

三层记忆模型：

```
第一层：工作记忆    → memory/YYYY-MM-DD.md（日常笔记，原始事件）
第二层：长期记忆    → MEMORY.md（精炼洞察， distilled 知识）
第三层：知识索引    → memory/index.md（可搜索的主题/人物/项目索引）
```

## 核心操作

### 1. 编码 — 写下来

绝不依赖"心理笔记"。所有重要信息立即写入文件。

**日常笔记**（`memory/YYYY-MM-DD.md`）：
- 事件、决策、对话的原始日志
- 时间戳关键动作：`### HH:MM - 事件`
- 标记重要条目：`!important`、`!todo`、`!decision`
- 全天持续追加，不要等到总结时才写

**长期记忆**（`MEMORY.md`）：
- 从日常笔记中精炼的洞察
- 按主题分节组织（`##` 标题）
- 包含：人物偏好、项目状态、经验教训、模式
- 在心跳期间或被明确要求时更新

### 2. 检索 — 智能搜索

检索记忆时使用分层方法：

1. 先用 **memory_search** 进行聚焦查询
2. 用 **memory_get** 拉取结果中的特定行范围
3. 如果知道日期，直接读取当天的文件
4. 查看 `memory/index.md` 获取主题导航

**搜索策略：**
- 使用语义查询：`memory_search("用户时区偏好")` 而非 `memory_search("时区")`
- 交叉引用：在 MEMORY.md 中搜索上下文，然后在日常文件中查找细节
- 时间查询：如果知道大致日期，直接读取那天的文件

### 3. 巩固 — 日常提炼为长期

每隔几天（在心跳或空闲时）：

1. 阅读最近的 `memory/YYYY-MM-DD.md` 文件
2. 提取：决策、教训、偏好、项目更新
3. 合并到 `MEMORY.md` 的相关章节
4. 从 MEMORY.md 中删除过时信息
5. 更新 `memory/index.md` 添加新主题/人物

**巩固规则：**
- 提炼，不要复制——提取精髓
- MEMORY.md 保持在 200 行以内；积极修剪
- 归档已完全整合的旧日常文件
- 当 MEMORY.md 过大时，拆分为 `memory/topics/*.md`

### 4. 关联 — 知识索引

维护 `memory/index.md` 作为轻量主题索引：

```markdown
## 人物
- 大凯 → MEMORY.md#L5, memory/2026-03-27.md

## 项目
- 财务自由 → MEMORY.md#L12, memory/2026-03-26.md

## 偏好
- 沟通风格: 简洁, 不说废话 → MEMORY.md#L8
```

在巩固期间更新索引。保持精简——只收录有多个引用的条目。

## 记忆生命周期

```
捕获 → 组织 → 提炼 → 关联 → 修剪
  ↓      ↓       ↓       ↓       ↓
日常笔记  标签+   合并到   更新    归档旧
       时间戳  MEMORY.md 索引    文件
```

**捕获：** 将所有重要内容写入日常笔记。
**组织：** 使用标签（`!important`、`!todo`、`!decision`）和时间戳。
**提炼：** 定期将日常内容合并到 MEMORY.md。
**关联：** 用交叉引用更新知识索引。
**修剪：** 从 MEMORY.md 中删除过时条目；归档已整合的日常文件。
