# Kuafu — 进化模式与高级策略

## 规则冲突检测

添加新规则时检查与现有规则的冲突：

1. 读取 `memory/kuafu-rules.md`
2. 检查同一类别下的所有规则
3. 如果新规则与现有规则矛盾：
   - **如果新规则更准确**：替换旧规则，记录变更
   - **如果旧规则仍然正确但需要细化**：修改旧规则而非添加新规则
   - **如果两者在不同场景下都有效**：为每条规则添加场景限定

## 规则效果评分

维护规则一段时间后，评估其有效性：

```markdown
### workflow Rule #3: 提交前检查  ⭐⭐⭐
- **触发次数：** 15
- **有效次数：** 14 (93%)
- **上次触发：** 2026-04-28
```

**评分标准：**
- ⭐⭐⭐ 高 — 频繁触发，几乎总有帮助。保留。
- ⭐⭐ 中 — 偶尔触发，大多有帮助。观察。
- ⭐ 低 — 很少触发或经常误判。考虑删除。

每月运行效果评估。删除或修订 30 天内未触发且低于 ⭐⭐ 的规则。

## 跨会话学习聚合

当模式跨越多个会话时：

1. 检查最近的日常记忆文件（过去 7 天）
2. 寻找重复主题：同类错误、同类用户纠正、同类工具问题
3. 如果同一模式在不同会话中出现 3+ 次 → 这是系统性问题 → 创建规则
4. 如果正向模式出现 3+ 次 → 将其编码为最佳实践

**聚合示例：**
```
会话 1: 忘记提交修改
会话 3: 又忘记提交修改
会话 5: 第三次忘记提交修改
→ 规则: 编辑后始终提交工作区变更 (workflow → #7)
```

## 从零启动

首次运行夸父（无现有规则文件）：

1. **自我评估**已知弱点：
   - 哪类错误最常发生？
   - 哪些用户纠正最频繁？
   - 哪些工作流感觉笨拙？
2. **创建 3-5 条初始规则**
3. **初始化指标文件**，计数器归零
4. **在当天的记忆文件中记录启动事件**

不要一开始就创建 50 条规则。从小开始，让系统有机生长。

- **操作级变更** → 提升到 `AGENTS.md`（女娲的工作区记忆）
